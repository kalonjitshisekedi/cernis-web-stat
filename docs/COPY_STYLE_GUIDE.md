# CERNIS copy style guide

## Language

Use **British English** throughout:
- colour (not color)
- optimise (not optimize)
- organisation (not organization)
- centre (not center)
- programme (not program, unless referring to software)
- licence (noun), license (verb)

## Headings

Use **sentence case** for all headings and navigation labels:

**Correct:**
- Our services
- How we work
- Case studies
- About us

**Incorrect:**
- Our Services
- How We Work
- Case Studies
- About Us

Only capitalise proper nouns (CERNIS, Azure, AWS, Microsoft, etc.).

## Tone

- **Professional**: Appropriate for B2B enterprise clients
- **Outcome-led**: Focus on results, not features
- **Concise**: Avoid jargon and unnecessary words
- **Confident**: State capabilities clearly without overselling

## Key messages

Always emphasise these outcomes:
1. **Cost reduction** - cloud optimisation, automation, reduced waste
2. **Operational effectiveness** - faster processes, better decisions, improved reliability

## Brand name usage

- **CERNIS** - Use in headlines, navigation, and formal contexts
- **Cernis** - Acceptable in running text for readability

## Call-to-action text

Use clear, action-oriented language:
- "Start a conversation" (preferred for contact)
- "Learn more"
- "View our work"
- "Get in touch"

Avoid generic phrases like "Click here" or "Submit".

## Numbers and dates

- Spell out numbers one to nine; use numerals for 10 and above
- Use day-month-year format: 17 December 2025
- Use 24-hour time or specify am/pm
